🔧 How Guardian DNS Works

Guardian DNS uses a pure-Python DNS proxy (run_dns_proxy.py) that intercepts every DNS query on your machine. Blocked domains resolve to 0.0.0.0 (unreachable). Every query goes through 3 stages:

1
DB Check (instant) — Already-decided domains are answered from SQLite in <1ms. permanent_block and temporary_block both return 0.0.0.0.
2
Seed Blocklist + Keyword Heuristics (instant) — 186 curated harmful domains pre-loaded at startup, plus 100+ regex patterns on domain names and TLDs. Blocks on the first visit without waiting for AI.
3
Forward + Local AI (Ollama) — Unknown domains are forwarded to 8.8.8.8 (page loads), then queued for llama3:8b in the background. Result updates the DB; proxy blocks on the next DNS query.

🤖 Ollama Setup (Required for AI Classification)

Guardian DNS uses Ollama to run an LLM locally on your machine. Here's how to get it running:

1
Install Ollama — Download from ollama.com or via Homebrew:
brew install ollama
2
Start the Ollama service
ollama serve

Runs on http://127.0.0.1:11434 by default. Keep it running in a terminal tab.

3
Pull a model — We recommend llama3:8b for the best accuracy/speed balance on Apple Silicon:
ollama pull llama3:8b

For faster but less accurate results: ollama pull llama3.2:3b
For highest accuracy: ollama pull llama3:70b (requires ~40GB RAM)

4
Verify Ollama is connected — Check the Reviewer Console header for the green Ollama status badge, or:
curl http://127.0.0.1:11434/api/tags

🟢 Start the DNS Proxy

Starts the pure-Python DNS proxy on port 53 and points your Mac's DNS at 127.0.0.1. Run these commands in order (Tab 2 = proxy, Tab 3 = DNS switch).

Step 1 — Free port 53 (if dnsmasq is running):

sudo brew services stop dnsmasq

Step 2 — Start the proxy (keep this terminal open):

sudo .venv/bin/python3 run_dns_proxy.py

Step 3 — Point Wi-Fi DNS at the proxy:

networksetup -setdnsservers Wi-Fi 127.0.0.1
Verify: dig +short pornhub.com @127.0.0.1 → should return 0.0.0.0
Tip: run Guardian server first (uvicorn app.main:app --host 127.0.0.1 --port 8000), then the proxy, then Step 3.

🔴 Stop Filtering & Restore DNS

Stops the DNS proxy and restores your Mac's automatic (DHCP) DNS settings. Run in any terminal.

Step 1 — Restore DNS to automatic:

networksetup -setdnsservers Wi-Fi empty

Step 2 — Stop the proxy:

Ctrl-C in the proxy terminal
After Step 1 your Mac uses DHCP DNS and Guardian no longer intercepts queries.
